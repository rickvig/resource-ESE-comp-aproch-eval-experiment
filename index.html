Supplementary Material for the Journal Manuscript Submission (ID JSS-D-17-00677)


VMTools-RA: A Reference Architecture for Software Variability Tools

Ana Paula Allian¹, Edson OliveiraJr¹, Bruno Sena², Elisa Yumi Nakagawa²
State University of Maringá (UEM)¹; University of São Paulo (USP)²
ana.allian[at]usp[dot]br; edson[at]din[dot]uem[dot]br; brunosena[at]usp[dot]br; elisa[at]icmc[dot]usp[dot]br




Paper Abstract.  CONTEXT: Most software systems need to support an intensive amount of variability, i.e., distinct locations in the software where behavior can be configured. This trend requires support of tools. MOTIVATION: Nowadays, several tools have been developed to deal with this variability in software systems. However, no single solution offers all desired functionalities required by industrial practitioners. Consequently, they have to create their own tools and combine different means to manage such variations without following established patterns and knowledge about variability management processes. OBJECTIVE: The major goal of this study is to present the specification of VMTools-RA reference architecture. It encompasses knowledge and practice for building, evolving, and providing interoperability among these tools. METHOD: For creating VMTools-RA, we adopted the ProSA-RA process, which encourages the systematic design and evaluation of reference architectures. We evaluate VMTools-RA in twofold with a: i) qualitative empirical study based on Grounded Theory procedures applied to experts in reference architecture and software variability; and ii) controlled experiment. RESULTS: Results from the qualitative study allowed us to improve our reference architecture. Results from the experiment provided preliminary evidence VMTools-RA is an effective and efficient reference architecture for designing tools to deal with software variability. In addition, we provided an illustrative example of instantiation for using VMTools-RA. CONCLUSION: We conclude VMTools-RA provides evidence that it is a feasible reference architecture for developing and improving software variability tools. We believe VMTools-RA could benefite the adapting, developing, and/or interoperating variability tools in industrial settings.
